\doxysubsection{Detailed Description}
Direct Memory Access Driver. 

~\newline
 \hypertarget{group__dmadrv_dmadrv_intro}{}\doxysubsection{\texorpdfstring{Introduction}{Introduction}}\label{group__dmadrv_dmadrv_intro}
The DMADRV driver supports writing code using DMA which will work regardless of the type of the DMA controller on the underlying microcontroller. Additionally, DMA can be used in several modules that are completely unaware of each other. The driver does not preclude use of the native emlib API of the underlying DMA controller. On the contrary, it will often result in more efficient code and is necessary for complex DMA operations. The housekeeping functions of this driver are valuable even in this use-\/case.

The \doxylink{dmadrv_8c}{dmadrv.\+c} and \doxylink{dmadrv_8h}{dmadrv.\+h} source files are in the emdrv/dmadrv folder.

\begin{DoxyNote}{Note}
DMA transfer completion callback functions are called from within the DMA interrupt handler.
\end{DoxyNote}
~\newline
 \hypertarget{group__dmadrv_dmadrv_conf}{}\doxysubsection{\texorpdfstring{Configuration Options}{Configuration Options}}\label{group__dmadrv_dmadrv_conf}
Some properties of the DMADRV driver are compile-\/time configurable. These properties are stored in a file named \doxylink{dmadrv__config_8h_source}{dmadrv\+\_\+config.\+h}. A template for this file, containing default values, is in the emdrv/config folder. Currently the configuration options are as follows\+: \begin{DoxyItemize}
\item The interrupt priority of the DMA peripheral. \item A number of DMA channels to support. \item Use the native emlib API belonging to the underlying DMA hardware in combination with the DMADRV API.\end{DoxyItemize}
Both configuration options will help reduce the driver\textquotesingle{}s RAM footprint.

To configure DMADRV, provide a custom configuration file. This is an example \doxylink{dmadrv__config_8h_source}{dmadrv\+\_\+config.\+h} file\+:
