\doxysubsection{Detailed Description}
Typedef to store the segment data of a 14-\/segment alphanumeric character. 

The typedef uses a 16 bit integer to store the data. As we need only 14 segments, the upper two bits are unused. The bits are mapped to the segments in the following way\+:

-\/-\/-\/-\/-\/-\/--- 0,a -\/-\/-\/-\/-\/---

\texorpdfstring{$\vert$}{|} \textbackslash{}7,h \texorpdfstring{$\vert$}{|}8,j / \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|}5,f \textbackslash{} \texorpdfstring{$\vert$}{|} /9,k \texorpdfstring{$\vert$}{|}1,b \texorpdfstring{$\vert$}{|} \textbackslash{} \texorpdfstring{$\vert$}{|} / \texorpdfstring{$\vert$}{|}

--- 6,g -- -- 10,m --

\texorpdfstring{$\vert$}{|} / \texorpdfstring{$\vert$}{|} \textbackslash{}11,n \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|}4,e /13,q \texorpdfstring{$\vert$}{|}12,p \textbackslash{} \texorpdfstring{$\vert$}{|}2,c \texorpdfstring{$\vert$}{|} / \texorpdfstring{$\vert$}{|} \textbackslash{} \texorpdfstring{$\vert$}{|}

-\/-\/-\/-\/-\/-\/--- 3,d -\/-\/-\/-\/-\/---

The bits can be accessed in two ways\+:


\begin{DoxyEnumerate}
\item Using a single, 16 bit value (raw).
\item Accessing individual bits using bit fields (a..q). 
\end{DoxyEnumerate}
